# 更新记录

## 1. 新增文件描述功能

- **后端**: `FileController.java` 新增 `@PutMapping("/description/{id}")` 接口，通过 `@RequestParam` 接收 description，更新文件描述用于搜索
- **前端 API**: `react-suiqu/src/api/file.js` 新增 `updateDescription(id, description)` 函数，发送 `PUT /file/description/{id}?description=...`
- **前端 UI**: `react-suiqu/src/pages/FilePage.jsx`
  - 文件操作栏新增"添加描述"按钮（`EditOutlined` 图标），仅非文件夹显示
  - 点击弹出 Modal，内含 `Input.TextArea` 用于输入描述
  - 支持编辑已有描述（预填当前值）
  - 保存后自动刷新文件列表

## 2. 修复文件列表列字段不匹配

后端 `FileVO` 返回的字段名与前端 `dataIndex` 不一致，导致文件名、创建时间不显示：

| 字段 | 原值 | 修正值 |
|------|------|--------|
| 文件名 `dataIndex` | `name` | `fileName` |
| 创建时间 `dataIndex` | `updateTime` | `createTime` |
| 目录判断 | `record.type === 'DIRECTORY'` | `record.isDir === 1` |
| 图标函数参数 | `getFileIcon(type)` | `getFileIcon(record)` |
| 双击打开目录 | `record.name` | `record.fileName` |

## 3. 修复创建时间显示格式

后端 Jackson 默认将 `LocalDateTime` 序列化为数组 `[2026,6,6,17,7,12]`，新增 `formatTime` 函数统一格式化为 `YYYY-MM-DD HH:mm:ss`

## 4. 修复搜索页面

- `react-suiqu/src/pages/SearchPage.jsx`:
  - 文件名 `dataIndex` 由 `name` 修正为 `file_name`（后端 JSON 返回 snake_case）
  - 新增 `formatTime` 格式化创建时间
  - 移除对 `FileIndex` 不存在的 `type`/`fileType` 字段的引用，图标统一使用 `FileOutlined`
- **后端问题 `FileIndex.java`（未修改，需注意）**：
  1. Jackson 序列化 `name` 字段变为 `file_name`、`userId` 变为 `user_id`，说明存在全局 snake_case 命名策略或 `@JsonProperty` 配置导致前后端字段名不匹配
  2. Elasticsearch 未安装/未运行 — `SearchController` 调用 `esOperations.search()` 会抛异常
  3. Canal + RabbitMQ 同步未启动 — `FileSyncConsumer` 未消费 binlog，ES 索引中无数据
  4. ES 索引 `files` 未创建 — 需确认 Spring Data ES 能自动创建或手动创建 mapping
  5. 已有数据未包含 `description` 字段 — 新增字段后需对已有文档重建索引才能搜索到描述
